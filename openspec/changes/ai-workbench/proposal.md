# Proposal: AI 工作台

## Why

产品经理的日常工作中，信息碎片化严重——不懂的名词随手记在微信/备忘录、分散各处忘记查询；TODO 淹没在列表里不知道今天该干什么；业务反馈转给开发后忘记追问；每天手动写日报、每周汇总周报耗时。需要一个统一的 AI 驱动工作台，解决“记了但忘了、记了但找不到、没有 follow-up 机制”三大痛点。

## What Changes

- **新增**：知识库模块，支持5种类型知识卡片（术语/方案/灵感/纪要/问题），正方形卡片网格布局，待了解/已了解状态一目了然，支持搜索筛选 CRUD
- **新增**：日程表模块，周视图日历 + 每日详情，支持 TODO 优先级管理（必须解决/重点关注/普通）
- **新增**：遗留问题模块，未解决问题持续显示，设置计划解决时间，逾期红色高亮
- **新增**：委托跟进模块，转给开发后自动设置追问提醒（默认3小时后+下班前），追问时间线可视化，状态流转（等待中→已追问→已回复→已解决）
- **新增**：日报/周报自动生成，每天18:00自动生成日报，每周五18:00自动生成周报（沿用现有周报模板），支持手动触发
- **新增**：AI 自然语言对话，支持打字/语音输入，自动意图识别路由（新增名词/任务/委托/查询/生成报告），多意图自动拆解执行
- **新增**：系统设置，支持配置 AI 模型 API（提供商/Key/模型/Base URL）
- **新增**：PC + 移动端双端支持，响应式布局 + PWA 安装到桌面，云端部署

## Capabilities

### New Capabilities

- `knowledge-base`: 知识卡片管理——5种类型卡片（术语/方案/灵感/纪要/问题），正方形网格布局，待了解/已了解状态区分，搜索筛选，关联卡片，编辑历史记录
- `task-management`: 任务与日程管理——TODO 优先级管理（必须解决/重点关注/普通），周视图日历，每日详情，遗留问题跟踪（计划解决时间+逾期标记）
- `delegation-tracking`: 委托跟进——转交开发后自动设置追问提醒（默认规则+用户自定义），追问时间线可视化，状态流转（等待中/已追问/已回复/已解决），结论记录
- `auto-reports`: 日报周报自动生成——活动自动记录，日报每天18:00自动生成（完成任务+新增名词+会议+反馈），周报每周五18:00自动生成（沿用现有模板：核心摘要/项目进展/具体产出/会议沟通/学习成长/建议/下周计划），支持手动触发
- `ai-conversation`: AI 自然语言对话——打字/语音输入，意图识别（新增名词/新增任务/新增委托/查询知识库/查今日概览/生成报告），多意图自动拆解，内嵌结果卡片展示
- `system-config`: 系统配置——AI 模型 API 配置弹窗（提供商/Key/模型/Base URL），测试连接，加密存储

## Impact

- **新增项目**：Next.js 全栈应用（App Router + Prisma + PostgreSQL）
- **部署**：Vercel / Railway 云端部署，单一 URL
- **AI 依赖**：Claude API（Anthropic）
- **PWA 依赖**：Service Worker + Web Push API + manifest.json
- **定时任务**：Vercel Cron Jobs（日报生成、周报生成、追问扫描、每日提醒）
- **数据存储**：PostgreSQL（Supabase 或 Railway 内置）
