> 版本：v1.0 | 日期：2026-07-06 | 作者：郭红军

## 1. 产品概述

### 1.1 产品定位

AI 工作台是一款面向产品经理的个人效率工具，解决日常工作中“信息碎片化、跟进遗漏、汇报耗时”三大痛点。通过 **自然语言驱动的聊天交互 + 结构化仪表盘**，实现知识管理、任务跟进、日程规划、自动汇报的一站式闭环。

### 1.2 目标用户

- **首要用户**：郭红军（产品经理），个人使用
- **未来扩展**：团队共享使用

### 1.3 核心价值

| 痛点 | 现状 | 解决方案 |
|------|------|----------|
| 不懂的名词随手记录，分散在各处，后续忘记查询 | 微信文件传输助手 / 手机备忘录 | 统一名词库，待了解/已了解状态一目了然，支持搜索 |
| 备忘录、灵感碎片散落各地，容易遗忘 | 同上 | 知识卡片统一管理，多类型支持 |
| TODO 淹没在列表里，不知道今天该干什么 | 各种 TODO 工具 | 智能日程表 + 每日重点提醒 |
| 业务反馈转给开发后，忘记追问 | 靠脑记 | 委托跟进模块，自动定时提醒追问 |
| 每天手动写日报，每周汇总周报耗时 | 手动撰写 | 自动记录活动 → 日报 → 周报全自动生成 |

## 2. 使用场景与平台

### 2.1 使用场景

```
┌─────────────────────────────────────────────────────────────────┐
│  工作日白天（PC 端）               碎片时间（手机端）              │
│  ──────────────────               ──────────────                 │
│  • 键盘打字输入                    • 语音输入为主                  │
│  • 仪表盘全面操作                  • 快速查看/查询                │
│  • 增删改查各模块                  • 添加名词/TODO                 │
│  • 查看日报/周报                   • 响应追问提醒                  │
│  • 处理委托跟进                    • 浏览日程                      │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 平台方案

- **技术选型**：Next.js 全栈应用（响应式布局 + PWA）
- **部署**：云端（Vercel / Railway），单一 URL
- **PC 端**：浏览器访问，完整仪表盘体验
- **手机端**：同一 URL，响应式布局 + PWA 安装到桌面，接近原生 App 体验
- **AI 能力**：对接 Claude API，实现自然语言理解和内容生成

## 3. 功能模块

### 3.0 全局：AI 自然语言交互

**位置**：仪表盘内嵌聊天面板 + 各模块页面内快捷入口

**能力**：用户通过打字或语音输入自然语言，AI 自动识别意图并路由到对应模块

```
意图识别映射：
┌─────────────────────────────────────────────────────────────────┐
│ 用户输入                                          → 系统动作      │
├─────────────────────────────────────────────────────────────────┤
│ "K8s 不太懂，记一下"                              → 知识库：新增名词卡片（待了解）    │
│ "矢量数据库是专门存高维向量的..."                  → 知识库：新增名词卡片（已了解+解释）│
│ "登录页需要支持手机号+验证码登录"                  → 知识库：新增方案卡片              │
│ "明天下午3点评审方案"                             → 日程：新增TODO+设置提醒           │
│ "李老板反馈登录页报错了，转给王工排查"            → 委托：新增委托任务+自动设提醒      │
│ "今天下班前提醒我追问王工那个bug"                 → 委托：设置追问提醒                │
│ "今天有什么重点？"                                → 查询：返回今日概览                │
│ "K8s 查到了吗？"                                  → 查询：搜索名词库                  │
│ "帮我写周报"                                      → 周报：生成本周周报                │
│ "GraphQL 是啥来着？"                              → 查询：搜索已了解名词的解释         │
└─────────────────────────────────────────────────────────────────┘
```

### 3.1 知识库

#### 3.1.1 知识卡片类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 📝 术语 | 不懂的专业名词 | K8s、GraphQL、向量数据库 |
| 📐 方案 | 产品需求方案设计 | 登录页重构方案、数据看板优化方案 |
| 💡 灵感 | 碎片化的想法/灵感 | “数据看板可以加个预警功能” |
| 📋 纪要 | 会议/沟通记录 | 7/6 设计评审会纪要 |
| ❓ 问题 | 需要搞清楚的业务逻辑/政策/开发实现 | “为什么订单超时取消是30分钟？” |

#### 3.1.2 名词卡片字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | ✅ | 名词/方案名称 |
| 类型 | 枚举 | ✅ | 术语 / 方案 / 灵感 / 纪要 / 问题 |
| 状态 | 枚举 | ✅ | 待了解 / 已了解（仅术语类型显示） |
| 我的理解 | 富文本 | ❌ | 自己的解释或方案内容 |
| 关键概念 | 标签数组 | ❌ | 关联的关键词 |
| 关联卡片 | 引用数组 | ❌ | 关联的其他卡片 |
| 来源 | 文本 | ❌ | 在哪里遇到的 |
| 创建时间 | 时间戳 | 自动 | |
| 更新时间 | 时间戳 | 自动 | |
| 历史记录 | 数组 | 自动 | 每次编辑的时间戳记录 |

#### 3.1.3 知识库功能

- ✅ 列表视图，支持按类型筛选、按状态筛选（待了解/已了解）、关键词搜索
- ✅ 卡片详情页，支持编辑、删除
- ✅ 待了解 vs 已了解：红色标记 vs 绿色标记，列表即可区分
- ✅ 待了解名词数量达到阈值时提醒（如≥5个未了解）
- ✅ 通过自然语言新增（聊天 + 语音）
- ✅ 移动端：快速查看、语音新增、搜索
- ✅ PC 端：正方形卡片网格布局（3列），移动端（2列）
- ✅ 卡片按类型用不同颜色标识（术语灰、方案蓝、问题橙、灵感蓝、纪要灰）

#### 3.2.1 TODO 字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|

| 截止日期 | 日期 | ❌ | |
| 状态 | 枚举 | ✅ | 待处理 / 进行中 / 已完成 |
| 关联卡片 | 引用 | ❌ | 关联知识库卡片 |

| 完成时间 | 时间戳 | 自动 | |

#### 3.2.2 日程表视图

- 每天早上 09:00 推送今日重点提醒

### 3.3 遗留问题

#### 3.3.1 遗留问题字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | ✅ | 问题描述 |
| 计划解决时间 | 日期 | ❌ | 预计何时解决 |
| 状态 | 枚举 | ✅ | 待解决 / 已解决 |
| 标签 | 文本 | ❌ | 分类标签（如：需求对齐、技术对接、基础设施） |
| 创建时间 | 时间戳 | 自动 | |
| 解决时间 | 时间戳 | 自动 | |

#### 3.3.2 遗留问题规则

- 未解决前持续显示，不会自动消失
- 超过计划解决时间 → 标记为「已逾期」，红色高亮
- 勾选后标记为「已解决」，移入已解决列表
- 与日程表在同一页面，通过标签页切换「日程表」和「遗留问题」

### 3.4 委托跟进

#### 3.3.1 委托任务字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | 文本 | ✅ | 委托事项描述 |
| 转交给 | 文本 | ✅ | 被委托人（如"王工"） |
| 来源 | 文本 | ❌ | 谁反馈的（如"李老板"） |
| 状态 | 枚举 | ✅ | 等待中 / 已回复 / 已解决 |
| 追问时间 | 时间 | 自动+手动 | 默认3小时后+下班前，用户可自定义 |
| 自定义追问 | 文本 | ❌ | 如"明天上午10点提醒我" |
| 优先级 | 枚举 | ✅ | 同 TODO 优先级 |
| 结论 | 文本 | ❌ | 开发回复的结论 |
| 创建时间 | 时间戳 | 自动 | |
| 解决时间 | 时间戳 | 自动 | |

#### 3.3.2 追问规则

```
默认规则（用户可覆盖）：
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  转出时间          首次追问              二次追问                │
│  ────────          ────────              ────────                │
│  上午 (9:00-12:00)  3小时后               当天 18:00             │
│  下午 (12:00-17:00) 3小时后               当天 18:00             │
│  17:00后            次日 10:00            次日 18:00             │
│                                                                 │
│  用户自定义优先级 > 默认规则                                      │
│  例如："2小时后提醒我" → 覆盖默认3小时                            │
│  例如："明天再问" → 跳过默认，次日提醒                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.3.3 委托交互

- 委托卡片默认展开，显示追问时间线（转出 → 首次追问 → 二次追问 → 解决）
- 已解决委托默认折叠，点击「查看详情」展开
- 操作按钮：已追问、记录结论（弹窗输入结论+状态）、修改追问时间、删除
- 状态流转：等待中 → 已追问 → 已回复 → 已解决
- 状态标记用不同颜色区分：等待中（橙色）、已回复（蓝色）、已解决（绿色）

#### 3.3.4 委托提醒

- 到追问时间 → 推送通知（PC + 手机）
- 通知内容：委托标题 + 转交给谁 + 已经过了多久
- 用户可标记：已追问 / 已回复 / 稍后提醒

### 3.5 日报 & 周报

#### 3.4.1 活动自动记录

系统自动记录用户在工作台的所有活动，作为日报/周报的素材：

| 活动类型 | 记录内容 |
|----------|----------|
| 完成任务 | 任务标题、完成时间 |
| 新增名词 | 名词标题、类型 |
| 了解名词 | 从待了解→已了解，记录解释 |
| 新增委托 | 委托标题、转交给谁 |
| 委托结案 | 委托标题、结论 |
| 参加会议 | 会议纪要卡片 |
| 新增方案 | 方案卡片标题 |

#### 3.4.2 日报生成

- **触发时间**：每个工作日 18:00 自动生成
- **内容**：完成的任务 + 新记录的名词 + 参加的会议 + 处理/转出的反馈
- **格式**：简洁列表，按类型分组
- **存储**：自动存入数据库，可在日报页面查看历史
- **无需用户确认**：自动留存，作为周报素材

#### 3.4.3 周报生成

- **触发时间**：每周五 18:00 自动生成
- **格式**：沿用现有周报 Skill 模板

```
# 周报：YYYY年第N周（MM月DD日 - MM月DD日）

**汇报人：** 郭红军
**汇报周期：** YYYY年MM月DD日 – MM月DD日

---

## 🔑 核心摘要
（2-3句话，角色定位 + 关键成果/卡点）

## 🚀 项目 / 需求交付进展
（按项目-需求维度归总本周推动的进展，同类工作已汇总）

## ✅ 具体产出
（本周实际输出的交付物：文档、模板、原型、报告等）

## 📋 会议 & 沟通
（本周参与或主持的会议、评审、对接讨论）

## 📚 学习 & 成长
（从知识库自动提取：本周了解的新名词、新知识）

## 💡 建议
（从活动记录中提取的建议、反馈、风险提示）

## 📌 下周重点计划
（从日程表中提取下周 TODO，或根据本周进展推断）

---

（⚠️ 缺失提醒：仅当本周有工作日日报缺失时显示）
```

- **数据来源**：本周5天的日报 + 活动流
- **学习&成长**：自动从知识库中提取本周新增/了解的名词
- **下周计划**：自动从日程表中提取下周 TODO
- **存储**：保存为 Markdown 文件，可在周报页面查看、复制、编辑

#### 3.4.4 手动触发

- 用户可随时说“生成本周周报”手动触发周报生成
- 用户可随时说“生成今天日报”手动触发日报生成

## 4. 非功能需求

### 4.1 性能

- 页面首屏加载 < 3 秒
- API 响应 < 500ms（不含 LLM 调用）
- LLM 意图识别响应 < 3 秒

### 4.2 安全

- 初期个人使用，基于云端部署的基础认证
- 未来扩展团队时，增加用户认证和权限控制

### 4.3 数据

- 数据库：PostgreSQL（Supabase 或 Railway 内置）
- 数据归属：用户完全掌控
- 备份：云端自动备份

### 4.4 兼容性

- PC：Chrome / Edge / Safari 最新版
- 手机：iOS Safari 16.4+ / Android Chrome 最新版
- PWA：支持离线查看已缓存内容
