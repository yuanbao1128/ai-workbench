## 1. 产品概述

### 1.1 产品定位

AI 工作台是一款面向产品经理的个人效率工具，以 **“仪表盘为中枢、AI 聊天为贯穿线”** 的架构，实现知识管理、任务跟进、日程规划、自动汇报的一站式闭环。

**核心交互模型：**

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│   ┌──────────┐    ┌──────────────────┐    ┌──────────┐  │
│   │          │    │                  │    │          │  │
│   │  Sidebar │    │   Main Content   │    │  AI Chat │  │
│   │  224px   │    │   (flexible)     │    │  380px   │  │
│   │          │    │                  │    │  Fixed   │  │
│   │  📊 仪表  │    │  各模块主操作区    │    │  Panel   │  │
│   │  📝 知识  │    │                  │    │          │  │
│   │  📅 日程  │    │                  │    │  对话历史  │  │
│   │  📋 日报  │    │                  │    │  快捷建议  │  │
│   │  📊 周报  │    │                  │    │  输入框   │  │
│   │          │    │                  │    │          │  │
│   └──────────┘    └──────────────────┘    └──────────┘  │
│                                                          │
│   PC 端：三栏布局，AI 面板始终可见                          │
│   ──────────────────────────────────────                  │
│   手机端：单栏全屏，底部导航切换模块，AI 为独立 Tab         │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 1.2 目标用户

- **首要用户**：郭红军（产品经理），个人使用
- **未来扩展**：团队共享使用

### 1.3 v2.0 核心变更

| 变更项 | v1.0 | v2.0 |
|--------|------|------|
| **AI 聊天位置** | 仅仪表盘内嵌区域 | PC 端右侧 380px 固定面板，全局存在；手机端独立 Tab |
| **委托跟进** | 独立菜单页 `/delegation` | 取消独立页面，合并到仪表盘「待跟进」展开面板 |
| **待追问→待跟进** | 单一委托追问 | 统一 TODO 管理：委托追问 + AI 记录的 TODO + 手动添加 |
| **知识库布局** | 列表/卡片 | 3×3 紧凑正方形网格 + 前后端分页 |
| **日报布局** | 列表 | 3×3 网格卡片 + 分页 |
| **周报** | 报告中混合 | 独立导航页，专属周报模板 |
| **手机端 AI** | 无语音 | 专属对话 Tab，大号麦克风按钮，长按录音 |
| **底部导航** | 仪表/知识/日程/委托/日报 | 仪表/知识/日程/日报/AI |

## 4. 待跟进模块

### 4.1 定位

**待跟进**是一个统一的 TODO 管理中心，聚合两大数据来源：

```
┌─────────────────────────────────────────────────────────┐
│                    待跟进 = 统一 TODO 池                  │
│                                                         │
│  ┌─────────────────────┐    ┌─────────────────────┐    │
│  │  委托追问（Delegation）│    │  我的 TODO（Task）     │    │
│  │                     │    │                     │    │
│  │  • 转给别人的事项     │    │  • 自己要做的事       │    │
│  │  • 含追问时间线       │    │  • 含截止日期         │    │
│  │  • 状态：等待中/     │    │  • 状态：待处理/     │    │
│  │    已追问/已回复/    │    │    已完成           │    │
│  │    已解决           │    │  • 类型标记：TODO    │    │
│  │  • 默认追问规则      │    │                     │    │
│  └─────────────────────┘    └─────────────────────┘    │
│                                                         │
│  共同属性：                                              │
│  • 均出现在仪表盘「待跟进」面板                             │
│  • 均同步到日程表（如有日期）                               │
│  • 均支持 AI 自然语言创建                                  │
│  • 均支持推送提醒                                         │
└─────────────────────────────────────────────────────────┘
```

### 4.2 数据模型

#### 4.2.1 Task（新增 TODO 类型）

在原 Task 模型基础上增加 `type` 字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | 枚举 | ✅ | `SCHEDULE`（日程任务）/ `TODO`（待办事项） |
| title | 文本 | ✅ | 任务描述 |
| priority | 枚举 | ✅ | MUST / FOCUS / NORMAL |
| dueDate | 日期时间 | ❌ | 截止日期（有日期的 TODO 同步到日程表） |
| status | 枚举 | ✅ | PENDING / IN_PROGRESS / DONE |
| source | 文本 | ❌ | 来源（AI对话 / 手动添加 / 委托转TODO） |
| relatedCards | 引用数组 | ❌ | 关联知识库卡片 |
| createdAt | 时间戳 | 自动 | |
| completedAt | 时间戳 | 自动 | |

#### 4.2.2 Delegation（委托）

保持原有模型不变，在查询时与 TODO 聚合：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | 文本 | ✅ | 委托事项描述 |
| assignee | 文本 | ✅ | 被委托人 |
| source | 文本 | ❌ | 反馈来源 |
| status | 枚举 | ✅ | WAITING / ASKED / REPLIED / RESOLVED |
| priority | 枚举 | ✅ | MUST / FOCUS / NORMAL |
| followUpTimes | 时间数组 | 自动 | 追问时间点 |
| customFollowUp | 文本 | ❌ | 用户自定义追问时间 |
| conclusion | 文本 | ❌ | 结论 |
| timeline | JSON数组 | 自动 | 交互时间线 |
| createdAt | 时间戳 | 自动 | |
| resolvedAt | 时间戳 | 自动 | |

### 4.3 AI 自然语言创建

#### 4.3.1 入口

1. **仪表盘「待跟进」面板** → AI 快速输入框
2. **AI 聊天面板** → 直接对话
3. **全局浮动按钮**（移动端）

#### 4.3.2 意图识别规则

```
用户输入                                          → 系统动作
─────────────────────────────────────────────────────────────
"张三找我解决发电量报表问题，6.17前需要回复"     → 创建 TODO（type=TODO, dueDate=6/17）
"李老板反馈登录页报错，转给王工排查"              → 创建委托（assignee=王工, 自动算追问时间）
"今天要完成需求文档"                              → 创建 TODO（type=TODO, dueDate=today）
"周三前完成智能安全帽手册"                        → 创建 TODO（type=TODO, dueDate=周三）
"帮我把 K8s 搞清楚"                              → 创建 TODO（type=TODO, 关联K8s卡片）
"下午3点提醒我评审方案"                           → 创建 TODO（type=TODO, dueDate=today 15:00）
```

#### 4.3.3 AI 响应格式

**创建 TODO / 委托 — 成功后返回嵌入式卡片：**

```
┌──────────────────────────────────┐
│ ✅ 已记录待跟进事项：             │
│ ┌──────────────────────────────┐ │
│ │ 📌 发电量报表问题 → 回复张三   │ │
│ │ 📅 截止 6月17日              │ │
│ │ 已同步到日程表 ✓              │ │
│ └──────────────────────────────┘ │
└──────────────────────────────────┘
```

**记录术语 — 自动 AI 检索并生成知识卡片：**

```
┌───────────────────────────────────────────┐
│ 🔍 AI 已自动检索「K8s」并生成知识卡片：    │
│ ┌───────────────────────────────────────┐ │
│ │ [术语]  [🔴 待了解]                    │ │
│ │                                       │ │
│ │ K8s                                   │ │
│ │ Kubernetes（简称 K8s）是 Google 开源   │ │
│ │ 的容器编排平台…                        │ │
│ │                                       │ │
│ │ [容器] [编排] [云原生] [Google]         │ │
│ │                                       │ │
│ │ 📅 已记录 · 来源：AI 检索              │ │
│ │                                       │ │
│ │ [📖 查看详情] [✅ 标记已了解] [✏️ 编辑] │ │
│ └───────────────────────────────────────┘ │
└───────────────────────────────────────────┘
```

#### 4.3.4 AI 自动术语检索

当用户通过 AI 记录一个不懂的术语时，系统自动调用 AI 进行知识检索和初始化：

| 步骤 | 说明 |
|------|------|
| 1. 意图识别 | 识别为 ADD_TERM 意图 |
| 2. 创建卡片 | 创建类型=术语、状态=待了解的卡片骨架 |
| 3. **AI 检索** | 调用 AI 自动生成术语的定义、解释、关键标签 |
| 4. 填入内容 | 将 AI 生成的解释填入卡片 content 字段，标签填入 tags 字段 |
| 5. 呈现结果 | 在对话中返回嵌入式知识卡片，含 3 个操作按钮 |

**操作按钮：**
- **📖 查看详情** — 打开卡片详情模态框（跳转到知识库）
- **✅ 标记已了解** — 一键将状态从 UNKNOWN 改为 KNOWN
- **✏️ 编辑** — 打开编辑表单，可修改 AI 生成的内容

**设计原则：** AI 自动帮用户初始化名词解释，降低记录门槛。用户只需确认或微调，不需要从零写起。

### 4.4 日程同步规则

| 条件 | 同步行为 |
|------|----------|
| TODO 有 `dueDate` | 自动出现在日程表对应日期 |
| 委托有 `followUpTime` | 在追问时间点出现在日程表 |
| TODO 完成 | 日程表对应项自动标记完成 |
| 委托解决 | 日程表移除追问标记 |

### 4.5 交互操作

#### 4.5.1 委托追问操作

| 操作 | 效果 | 状态变更 |
|------|------|----------|
| **已追问** | 记录追问时间，状态变更 | WAITING → ASKED |
| **记录结论** | 弹出模态框，输入结论+状态 | ASKED → REPLIED / RESOLVED |
| **修改追问** | 弹出时间选择器，修改追问时间 | 更新 followUpTime |
| **删除** | 确认后删除（软删除） | - |

#### 4.5.2 TODO 操作

| 操作 | 效果 | 状态变更 |
|------|------|----------|
| **Checkbox 勾选** | 文字加删除线 + 透明度降至 55% + 背景变灰 → Toast "✅ 任务已完成" | PENDING → DONE |
| **取消勾选** | 恢复原始样式（删除线、透明度、背景还原） | DONE → PENDING |
| **编辑** | 弹出编辑表单 | - |
| **删除** | 确认后删除 | - |

> **设计原则：** 任务完成后保留在列表中（删除线 + 灰色），不删除、不消失。用户可看到已完成了什么，也可取消勾选恢复。

### 8.3 语音输入规格（手机端）

| 属性 | 说明 |
|------|------|
| 触发方式 | **长按**麦克风按钮开始录音，**松开**结束并发送 |
| 按钮样式 | 48×48px 圆形，蓝色渐变背景，白色麦克风图标 |
| 录音中样式 | 按钮变红（`#DC2626`），图标变 🔴，放大 1.15 倍 |

### 8.4 AI 对话功能

| 功能 | 说明 |
|------|------|
| 意图识别 | 自然语言 → 结构化意图 → 路由到对应模块 |
| 多意图 | 一段话中的多个意图可拆解分别执行 |
| 结果卡片 | 创建/查询成功后在对话中嵌入结构化卡片 |
| **术语自动检索** | 记录不懂的术语时，AI 自动检索定义、标签，生成知识卡片骨架，用户可直接查看、标记已了解或编辑 |
| **文件上传** | PC 端支持上传 .pdf/.docx/.doc/.txt/.md/.png/.jpg 文件，AI 自动解析并归档（会议纪要→纪要卡片、方案文档→方案卡片等） |
| 上下文记忆 | 保留最近 10 轮对话上下文 |
| 快捷建议 | 根据当前页面动态切换建议（仪表盘：记名词/添加TODO/转委托；知识库：记名词/查名词等） |
| 错误处理 | AI 不可用时提示"抱歉，AI 服务暂时不可用" |
| 思考中 | 发送后显示"思考中..."加载态 |

### 8.5 文件上传功能（PC 端）

#### 8.5.1 入口与交互

| 属性 | 说明 |
|------|------|
| 位置 | PC 端 AI 聊天面板输入框左侧，📎 图标按钮（虚线边框） |
| 样式 | 28×28px 图标按钮，虚线边框 `border-dashed`，hover 变实线 |
| 支持格式 | PDF、Word (.docx/.doc)、TXT、Markdown (.md)、PNG、JPG |
| 多选 | 支持一次选择多个文件 |
| 大小限制 | 单个文件 ≤ 10MB，总计 ≤ 30MB |

#### 8.5.2 上传后 AI 处理

```
用户上传文件 → AI 分析内容 → 根据内容智能归档：

┌──────────────────────────────────────────────────────┐
│ 文件类型            AI 行为                            │
│ ─────────           ────────                           │
│ 会议纪要 (.docx)     → 提取议题+决议 → 创建纪要卡片     │
│ 方案文档 (.pdf)      → 提取核心内容 → 创建方案卡片     │
│ 需求文档 (.docx)     → 提取需求要点 → 知识库存档       │
│ 截图 (.png/.jpg)    → OCR 识别文字 → 知识库记录        │
│ 未识别类型           → 提供选项：存档/转TODO/生成纪要   │
└──────────────────────────────────────────────────────┘
```

#### 8.5.3 响应格式

上传后 AI 在对话中返回：
- 文件名 + 大小确认
- 解析结果摘要
- 操作按钮：查看完整内容、编辑、存入知识库等

### 8.6 AI 对话跨页面行为

| 场景 | 行为 |
|------|------|
| 用户在仪表盘对话 | 对话保留在面板中 |
| 用户切换到知识库 | 面板保持，对话历史不丢失 |
| 用户在 AI 面板操作知识库 | 操作结果实时反映到主内容区 |
| 对话框关闭（移动端） | 对话历史保留，下次打开继续 |
