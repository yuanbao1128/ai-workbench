## 3. 组件规范

### 3.1 卡片（Card）

```
基础样式：bg-white rounded-xl border border-gray-200 shadow-sm
悬停样式：shadow-md border-gray-300（可选 hover 效果）

变体：
- 默认卡片：p-5（内边距 20px）
- 紧凑卡片：p-3（内边距 12px，用于列表项）
- 高亮卡片：bg-red-50 border-red-200（必须解决）
- 高亮卡片：bg-amber-50 border-amber-200（重点关注）
```

### 3.2 按钮（Button）

| 类型 | 样式 | 用途 |
|------|------|------|
| 主按钮 | `bg-primary-600 text-white rounded-lg px-4 py-2 font-medium` | 主要操作 |
| 幽灵按钮 | `text-gray-500 rounded-lg px-4 py-2 font-medium hover:bg-gray-100` | 次要操作 |
| 危险按钮 | `text-red-500` 幽灵按钮 | 删除等危险操作 |
| 图标按钮 | 无文字，仅图标 | 语音、通知等 |

### 3.3 徽章（Badge）

```
基础：inline-flex items-center px-2 py-0.5 rounded-full text-xs font-medium

状态变体：
- 待了解/必须解决：bg-red-50 text-red-600
- 已了解/已解决：bg-green-50 text-green-600
- 重点关注/等待中：bg-amber-50 text-amber-600
- 方案/灵感：bg-blue-50 text-blue-600
- 问题：bg-amber-50 text-amber-600
- 默认：bg-gray-100 text-gray-500
```

### 3.4 状态圆点（Dot）

```
- 待了解：w-2 h-2 rounded-full bg-red-500
- 已了解：w-2 h-2 rounded-full bg-green-500
```

### 3.5 标签（Tag）

```
样式：inline-flex items-center gap-1 px-2 py-0.5 rounded-md text-xs bg-gray-100 text-gray-600
用途：关键概念、关联卡片、名词列表
```

### 3.6 导航项（Nav Item）

```
默认：flex items-center gap-3 px-3 py-2.5 rounded-lg cursor-pointer text-sm text-gray-600
悬停：bg-gray-50
激活：bg-primary-50 text-primary-600 border-r-3 border-primary-600 font-semibold
徽章：ml-auto
```

### 3.7 优先级左边框

```
- 必须解决：border-l-3 border-red-600
- 重点关注：border-l-3 border-amber-600
- 普通：border-l-3 border-gray-200
```

### 3.8 输入框（Input）

```
样式：border border-gray-200 rounded-lg px-3 py-2 text-sm w-full
聚焦：border-primary-600 ring-3 ring-primary-600/10
```

### 3.9 聊天气泡

```
用户气泡：bg-primary-50 rounded-2xl rounded-br-sm px-3 py-2.5 max-w-[70%]
AI 气泡：bg-gray-50 rounded-2xl rounded-bl-sm px-3 py-2.5 max-w-[70%]
```

### 3.10 标签页切换（Tab）

```
默认：px-3.5 py-1.5 rounded-lg text-sm font-medium text-gray-500 cursor-pointer
激活：bg-primary-600 text-white
```

### 3.11 日历格子

```
默认：rounded-lg p-2 min-h-[80px] bg-gray-50 cursor-pointer
选中：rounded-lg p-2 min-h-[80px] bg-primary-50 border-2 border-primary-600
周末：bg-gray-100
```

### 3.12 模态框（Modal）

```
遮罩：fixed inset-0 bg-black/40 z-100 flex items-center justify-center
内容：bg-white rounded-2xl p-6 max-w-[640px] w-[90%] max-h-[80vh] overflow-y-auto shadow-2xl
```

## 4. 页面规格

### 4.1 仪表盘首页

- **路由**：`/`
- **布局**：2列网格（今日概览卡片）
- **内容**：必须解决、重点关注、待追问、待了解名词、AI 对话快捷入口
- **原型参考**：切换到「仪表盘」标签页

### 4.2 知识库列表

- **路由**：`/knowledge`
- **布局**：顶部筛选栏 + 垂直卡片列表
- **筛选**：类型（全部/术语/方案/灵感/纪要/问题）+ 状态（全部/待了解/已了解）+ 搜索框
- **列表项**：状态圆点 + 标题 + 类型徽章 + 状态徽章 + 摘要 + 时间
- **原型参考**：切换到「知识库」标签页

### 4.3 知识卡片详情

- **路由**：`/knowledge/:id`
- **布局**：单列，最大宽度 720px
- **内容**：标题 + 类型 + 状态 + 我的理解 + 关键概念 + 关联卡片 + 来源 + 历史记录 + 操作按钮
- **原型参考**：切换到「卡片详情」标签页

### 4.4 任务管理 & 日程表

- **路由**：`/tasks`
- **布局**：顶部日历周视图 + 下方当日详情
- **日历**：7列网格，可前后翻周
- **当日详情**：按优先级分组（必须解决/重点关注/普通）
- **原型参考**：切换到「日程表」标签页

### 4.5 委托跟进

- **路由**：`/delegation`
- **布局**：顶部筛选 + 垂直卡片列表
- **筛选**：全部/等待中/已解决
- **卡片**：标题 + 转交对象 + 来源 + 追问时间 + 优先级 + 操作按钮
- **原型参考**：切换到「委托跟进」标签页

### 4.6 周报查看

- **路由**：`/reports/weekly`
- **布局**：单列，最大宽度 720px
- **内容**：Markdown 渲染的周报 + 操作按钮
- **历史**：顶部可切换历史周报
- **操作**：生成本周、复制全文、编辑、下载 Markdown
- **原型参考**：切换到「周报」标签页

### 4.7 AI 对话

- **路由**：`/chat`
- **布局**：聊天界面（顶部标题栏 + 消息列表 + 底部输入框）
- **消息**：AI 气泡 + 用户气泡 + 内嵌卡片（委托确认等）
- **输入**：文本输入框 + 语音按钮 + 快捷建议
- **原型参考**：切换到「AI对话」标签页

## 6. 交互流程

### 6.1 用户日常使用流程

```
早上 9:00 收到推送 → 打开工作台 → 仪表盘查看今日概览
  ├── 点击具体模块处理 → 手动 CRUD
  └── AI 对话输入 → 自然语言操作

白天工作中：
  ├── 遇到不懂名词 → 聊天/语音输入 → 自动加入知识库
  ├── 收到业务反馈 → 聊天输入 → 自动创建委托+设提醒
  ├── 想到新灵感 → 聊天输入 → 自动加入知识库
  └── 完成 TODO → 手动勾选 / 聊天输入

下午 18:00 → 日报自动生成 → 存入数据库（无需确认）

周五 18:00 → 周报自动生成 → 可查看/编辑/复制
```

### 6.2 自然语言意图路由

```
用户输入 → Claude API 意图识别 → 实体提取
  ├── 新增名词 → 提取标题/解释 → 知识库创建卡片
  ├── 新增方案/灵感/纪要/问题 → 提取标题/类型/内容 → 知识库创建卡片
  ├── 新增 TODO → 提取标题/优先级/截止日期 → 任务创建
  ├── 转交/委托 → 提取事项/转交对象/来源/追问时间 → 委托创建+设提醒
  ├── 查询 → 提取对象 → 对应模块搜索返回
  ├── 生成日报/周报 → 提取时间范围 → 手动触发生成
  └── 未知 → 通用对话 AI 自由回答
```

### 6.3 委托追问提醒流程

```
创建委托 → 判断追问时间
  ├── 用户自定义时间 → 使用自定义时间
  └── 无自定义 → 默认规则：
      ├── 9:00-17:00 转出 → 3小时后首次追问 + 18:00二次追问
      └── 17:00后转出 → 次日10:00首次追问 + 次日18:00二次追问

定时任务每分钟扫描 → 到达追问时间 → 推送通知
用户操作：[已追问] [已回复] [稍后30分钟] [忽略]
```

### 6.4 日报→周报生成流程

```
每天 18:00 → 扫描当天活动流
  → 完成任务 + 新增名词 + 参加会议 + 处理/转出反馈
  → 生成日报 → 存入数据库

每周五 18:00 → 汇总本周5天日报
  → 合并同类项（同一项目/需求合并）
  → 提取量化数据
  → 从知识库提取本周学习内容
  → 从日程表提取下周计划
  → Claude API 润色（产品经理视角，语义忠实）
  → 生成周报 → 存入数据库
```

## 7. PWA 配置要点

### 7.1 manifest.json

```json
{
  "name": "AI 工作台",
  "short_name": "工作台",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#F8FAFC",
  "theme_color": "#2563EB",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192" },
    { "src": "/icon-512.png", "sizes": "512x512" }
  ]
}
```

### 7.2 离线缓存

- Service Worker 缓存关键页面和静态资源
- 仪表盘和知识库列表支持离线查看（已加载过的数据）
- 离线时显示“离线模式”标记

### 7.3 推送通知

- 使用 Web Push API
- 用户首次打开时请求通知权限
- 通知场景：委托追问、每日重点提醒、待了解名词阈值提醒
